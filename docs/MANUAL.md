# 営業手帳 操作マニュアル

Version: 1.0 Release Candidate

このマニュアルは利用者向けの操作説明です。Markdownで管理し、将来PDF化しやすいように章を独立させています。

## 1. ログイン

1. 営業手帳を開きます。
2. 登録済みのメールアドレスとパスワードを入力します。
3. ログイン後、ホーム画面が表示されます。
4. ログアウトは設定画面またはPC上部ヘッダーから実行します。

補足:

- Supabase設定がない環境ではログインできない場合があります。
- `.env` やAPIキーは共有しないでください。

## 2. ホーム

ホームでは、その日に確認すべき営業活動をまとめて見ます。

主な表示:

- 今日フォローすべき顧客
- 今週フォローすべき顧客
- ステータス別件数
- 通知カード
- 高スコア顧客

朝にホームを開き、期限切れフォローや通知カードを確認してから営業活動を始めます。

## 3. 顧客

顧客画面では取引先を一覧管理します。

できること:

- 顧客検索
- ステータス絞り込み
- タグ絞り込み
- 重要度絞り込み
- 顧客カルテを開く
- 営業メール作成へ進む

PCではテーブル中心、スマホではカード中心で表示します。

## 4. 担当者

担当者画面では会社に紐づく人物情報を管理します。

主な項目:

- 氏名
- 部署
- 役職
- メール
- 電話
- 決裁権
- 役職重要度
- 人物メモ

担当者情報は顧客カルテ、商談履歴、メール作成の参考情報として利用します。

## 5. 名刺

名刺画面では名刺画像とOCR結果を管理します。

操作:

1. 名刺を追加します。
2. スマホではカメラ、PCでは画像アップロードを使います。
3. OCR結果を確認します。
4. 必要に応じて編集します。
5. 担当者として保存します。

注意:

- OCR結果は必ず人が確認してください。
- 1社に複数枚の名刺を登録できます。

## 6. 商品

商品画面では商品マスターを管理します。

主な項目:

- 商品名
- カテゴリー
- メーカー
- 産地
- 温度帯
- 荷姿
- 原価
- 希望販売価格
- 粗利率
- 商品画像
- 商品資料
- スペックシート

原価と希望販売価格を入力すると、粗利率を確認できます。

## 7. 在庫

在庫は商品詳細画面から管理します。

主な項目:

- コスト
- 通貨
- 数量
- 単位
- 現物 / 先物
- 所有者
- 在庫ステータス
- ファーム期限
- ETA
- LOT
- 賞味期限
- メモ

在庫ステータス:

- フリー
- 案内中
- 出庫待ち
- 予約済
- ファーム
- 欠品
- 入港待ち
- 売切

見積作成時に在庫を参照し、粗利計算へ利用できます。

## 8. 仕入先

仕入先画面では国内仕入先と海外メーカーを管理します。

海外メーカーで管理する項目:

- 国
- FOB港
- Incoterms
- MOQ
- リードタイム
- 通貨
- 支払条件
- 温度帯
- 取扱商品

仕入先情報は商品、見積、在庫管理で参照します。

## 9. 見積PDF

顧客カルテから見積を作成できます。

操作:

1. 顧客カルテを開きます。
2. 見積履歴セクションで見積番号を入力します。
3. 商品、在庫、数量、単価、原価、有効期限、支払条件、納品条件を入力します。
4. PDFプレビューで内容を確認します。
5. PDFダウンロードまたは保存を実行します。

注意:

- 自動送信はしません。
- PDFはSupabase Storageへ保存し、DBにはURLとメタ情報を保存します。

## 10. サンプル

サンプル管理では、発送後の評価とフォローを管理します。

主な項目:

- サンプル名
- 発送日
- 到着予定日
- フォロー日
- ステータス
- 評価
- 次回アクション
- 追跡番号

サンプル評価待ちはホームやダッシュボードで確認します。

## 11. クレーム

クレームは削除せず、履歴として残します。

主な項目:

- クレーム種別
- 内容
- 発生日
- 対応状況
- 原因
- 再発防止策
- 対応期限
- 解決日
- 記載者

未解決クレームは顧客カルテやダッシュボードで目立つように確認します。

## 12. ダッシュボード

経営判断ダッシュボードでは、営業状況を集計します。

主な表示:

- 見積金額合計
- 見込粗利
- 粗利率
- ステータス別案件数
- 商品別見積額
- 顧客別見積額
- 在庫別引当状況
- フリー在庫
- ファーム在庫
- 出庫待ち
- サンプル評価待ち
- クレーム件数
- 期限切れフォロー

PCでは表、スマホではカードで確認します。

## 13. 検索

検索はPCヘッダー、顧客一覧、商品一覧などから利用できます。

検索対象例:

- 会社名
- 担当者
- 商品
- 仕入先
- タグ
- メモ
- 地域
- ステータス

検索で見つからない場合は、表記ゆれやタグの有無を確認してください。

## 14. バックアップ

設定画面からJSON Export / JSON Importができます。

Export:

- アプリ内データをJSONで出力します。
- Storageのファイル本体は含みません。
- URLとメタ情報のみ含みます。

Import:

- 既存データを消さず、同じIDは更新、ないIDは追加します。
- 取り込み前に現在のデータをExportしておくと安全です。

## 15. FAQ

### PCとスマホでデータが同期されません

両方で同じアカウントにログインしているか確認してください。設定画面で保存先がSupabaseになっているか確認し、「クラウドから再読み込み」を実行してください。

### 添付ファイルが保存できません

Supabase Storageの設定、ログイン状態、ネットワーク接続を確認してください。オフライン時はファイル本体をLocalStorageへ保存しません。

### Chrome拡張から追加できません

Chrome拡張が有効か、送信先が `https://eigyo-techo.vercel.app/import?companyName=` になっているか確認してください。

### AI生成が使えません

現在はダミー生成またはフォールバック生成を前提としています。外部APIキー未設定でも既存機能は動作します。

### 見積PDFが作れません

見積番号、顧客、商品、数量、単価などの必須情報を確認してください。Storage接続がない場合、保存処理でエラーになることがあります。
