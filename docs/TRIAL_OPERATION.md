# 1-2 Week Trial Operation Guide

営業手帳 Version 1.0候補を、1-2週間の実運用で確認するための手順書です。

目的は、実際の営業活動で使いながら「壊れていないか」「入力しやすいか」「PCとスマホで同期できるか」「次に直すべき点は何か」を記録することです。

## 運用前チェックリスト

- [ ] 本番Supabaseに最新migrationを適用した
- [ ] Vercelに以下の環境変数を設定した
- [ ] `VITE_SUPABASE_URL`
- [ ] `VITE_SUPABASE_PUBLISHABLE_KEY` または `VITE_SUPABASE_ANON_KEY`
- [ ] `VITE_OPENAI_API_KEY` は未設定でもテンプレート生成にfallbackすることを確認した
- [ ] `.env` や秘密情報をGitHubへ上げていない
- [ ] `npm.cmd run build` が成功する
- [ ] PCブラウザでログインできる
- [ ] スマホブラウザでログインできる
- [ ] PCで登録した取引先がスマホで見える
- [ ] スマホで登録した商談メモがPCで見える
- [ ] 名刺画像または添付ファイルをStorageへ保存できる
- [ ] JSON Exportを1回取得してバックアップできる

## 1-2週間の運用ルール

- 毎朝、Homeで今日のフォロー、期限切れ、通知を確認する
- 新しい会社は、取引先またはChrome拡張から登録する
- 外出先ではスマホの「追加」から名刺撮影、商談メモ、クレーム記録を優先して使う
- PCでは取引先一覧、商品一覧、顧客カルテで整理・編集する
- 商談履歴は後から直接書き換えず、追記・訂正として残す
- 添付ファイル本体はLocalStorageに保存しない
- 重要な操作前後は、必要に応じてJSON Exportを取得する

## Ver1.0 運用手順

1. 毎朝Homeを開き、通知、今日フォロー、期限切れを確認する
2. 外出先ではスマホ下部ナビの「追加」から、名刺撮影または商談メモを登録する
3. 会社情報の整理はPCの取引先一覧と顧客カルテで行う
4. 商品・見積・サンプルはPCでまとめて登録し、顧客カルテから紐づけを確認する
5. クレームは発生日、原因、対応期限、対応状況を必ず残す
6. 1日の終わりにPCで案件ステータスと次回フォロー日を更新する
7. 週1回、設定画面からJSON Exportを取得する
8. 週1回、このドキュメントの既知バグ・改善メモ・フィードバックを更新する

## 運用中に止める条件

- ログインできない
- PCとスマホで同じユーザーのデータが同期されない
- 取引先・商品・商談メモが保存できない
- Storage添付で重要ファイルが失われる
- RLSや権限設定により他ユーザーのデータが見える

上記が発生した場合は、実運用を止めてBUGFIX_RULE.mdに従い最小修正を行います。

## 次フェーズ判断

1-2週間の試験運用後、以下を判断します。

- Version 1.0として公開継続できるか
- 最優先で直すバグは何か
- スマホ現場入力で足りない項目は何か
- PC整理画面で足りない比較・検索機能は何か
- OpenAI / Gmail / Outlook / LINEの本番連携をどの順番で進めるか
