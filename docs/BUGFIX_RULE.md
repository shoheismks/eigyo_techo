# BUGFIX_RULE

## 基本方針
- まず原因を特定する
- 最小修正のみ行う
- 新機能追加は禁止
- 大規模リファクタは禁止
- 既存機能を壊さない
- PROJECT_RULE.md / ARCHITECTURE.md を優先する

## 修正手順
1. 再現条件を確認
2. 原因箇所を特定
3. 最小修正
4. npm run build
5. 修正内容と残課題を報告

## 確認対象
- PC表示
- スマホ表示
- ログイン
- Supabase保存
- Storage添付
- 入力フォーム
- 画面遷移

## 報告形式
- 原因
- 修正内容
- build結果
- 残課題
