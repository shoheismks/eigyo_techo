# 受注管理 Phase 4 納品書

## 目的

出荷済データから納品書を作成し、PDFとして保存・再発行できるようにする。

## 対象

- `delivery_notes`
- `delivery_note_lines`
- 納品書一覧
- 納品書詳細
- 納品書PDF
- 出荷から納品書作成
- 価格表示ON/OFF

## 作成ルール

- 納品書は `Shipped` の出荷データから作成する。
- 1つの出荷に対して有効な納品書は1件までとする。
- 作成時点の出荷、受注、顧客、発行元、明細情報をsnapshot保存する。
- 出荷後に商品マスターや受注内容が変わっても、作成済み納品書の内容は変えない。

## 価格表示

- 既定値はOFF。
- OFFの場合、PDFとプレビューに単価、金額、税、合計を表示しない。
- ONの場合のみ、単価、金額、税、合計を表示する。
- 設定は納品書ごとに保存し、再発行時も維持する。

## PDF

- A4縦。
- 20明細ごとに改ページする。
- 各ページに明細ヘッダーを再表示する。
- 顧客向けPDFに社内原価、粗利、営業利益、実質利益は表示しない。
- PDFは `app-attachments` Storage bucket に保存する。

## RLS

- `delivery_notes.user_id = auth.uid()`
- `delivery_note_lines.user_id = auth.uid()`
