# DATA_RULE

営業手帳のデータ入力ルール集です。

目的は、表記ゆれを減らし、検索・集計・引き継ぎ・AI生成の品質を安定させることです。

## 基本方針

- 1つの意味には1つの表記を使う。
- 会社名、担当者名、商品名は正式名称を優先する。
- 略称や社内呼称はメモ欄またはタグに記録する。
- 日付はアプリの日付入力を使い、手入力メモでは `YYYY-MM-DD` を推奨する。
- 数量、金額、単価、原価は数値として入力する。
- タグは短く、検索しやすい単語にする。
- 判断に迷う内容は削除せず、備考やメモに補足する。

## 会社名

### 入力ルール

- 法人名は正式社名を優先する。
- 株式会社、有限会社、合同会社などの法人格は原則として含める。
- 法人格の位置は登記名に合わせる。
- 店舗名や屋号だけで登録する場合は、会社備考に運営会社を記録する。
- 支店、営業所、店舗は会社名と別に、住所や備考で補足する。

### 推奨例

| 推奨 | 理由 |
| --- | --- |
| 株式会社山田食品 | 正式社名として検索しやすい |
| 山田食品株式会社 東京支店 | 支店まで識別できる |
| ABC FOODS Japan株式会社 | 英字表記を正式名に合わせている |

### 非推奨例

| 非推奨 | 理由 |
| --- | --- |
| 山田食品 | 法人格が抜けて重複しやすい |
| 山田 | 会社か担当者か判別しにくい |
| （株）山田食品 | 表記ゆれが起きやすい |

## 担当者名

### 入力ルール

- 姓と名の間は半角スペースを推奨する。
- 敬称は氏名欄に入れない。
- 部署、役職、決裁権は専用項目へ入力する。
- 読み仮名やニックネームは人物メモへ入力する。

### 推奨例

| 推奨 | 理由 |
| --- | --- |
| 山田 太郎 | 姓名が分かりやすい |
| 佐藤 花子 | 敬称なしで一覧表示しやすい |

### 非推奨例

| 非推奨 | 理由 |
| --- | --- |
| 山田様 | 敬称が混ざる |
| 山田部長 | 役職と氏名が混ざる |
| Yamada | 日本語名と混在し検索しにくい |

## 商品名

### 入力ルール

- 商品名は見積や資料に使える正式名称を入力する。
- 規格、グレード、カット形状、ブランドは必要に応じて商品名または説明欄に入れる。
- メーカー名、産地、温度帯、荷姿は専用項目へ入力する。
- 略称はタグまたはメモに入力する。

### 推奨例

| 推奨 | 理由 |
| --- | --- |
| 豪州産グラスフェッドビーフ サーロイン | 産地と商品内容が明確 |
| イタリア産モッツァレラチーズ 1kg | 規格が分かる |
| スモークベーコンスライス 500g | 形状と規格が分かる |

### 非推奨例

| 非推奨 | 理由 |
| --- | --- |
| 肉 | 商品を特定できない |
| チーズA | 他の商品と混同しやすい |
| サンプル用 | 商品マスター名として不明確 |

## 在庫

### 入力ルール

- 数量は数値で入力する。
- 単位は `kg`, `g`, `ケース`, `箱`, `袋`, `個` など既存候補を優先する。
- コストは通貨とセットで入力する。
- LOT、賞味期限、ETA、ファーム期限は分かる範囲で入力する。
- 在庫ステータスは候補から選ぶ。

### ステータス基準

| ステータス | 使用基準 |
| --- | --- |
| フリー | まだ顧客に引き当てていない |
| 案内中 | 顧客へ提案中だが確定していない |
| 出庫待ち | 出荷予定が決まっている |
| 予約済 | 顧客または案件に仮押さえしている |
| ファーム | 仕入・販売の確定期限がある |
| 欠品 | 案内できる在庫がない |
| 入港待ち | 輸入品などで到着待ち |
| 売切 | 販売済みで利用不可 |

### 推奨例

| 項目 | 推奨 |
| --- | --- |
| 数量 | 120 |
| 単位 | kg |
| 通貨 | JPY |
| LOT | LOT-202607 |

### 非推奨例

| 非推奨 | 理由 |
| --- | --- |
| だいたい100kg | 数値集計できない |
| たくさん | 数量が不明 |
| フリー？ | ステータスとして不確定 |

## 仕入先

### 入力ルール

- 仕入先名は正式名称を優先する。
- 海外メーカーは英字正式名を優先する。
- 国、通貨、Incoterms、FOB港、MOQ、リードタイムは分かる範囲で入力する。
- 担当者名、メール、電話は専用項目へ入力する。
- 支払条件や注意点はメモへ入力する。

### 推奨例

| 推奨 | 理由 |
| --- | --- |
| ABC Meat Co., Ltd. | 海外メーカーの正式名 |
| 株式会社東京フーズ | 国内仕入先の正式名 |
| Incoterms: FOB | 条件が明確 |

### 非推奨例

| 非推奨 | 理由 |
| --- | --- |
| ABC | 会社を特定しにくい |
| いつもの業者 | 引き継ぎできない |
| 海外の肉屋 | 検索・集計できない |

## 見積

### 入力ルール

- 見積番号は重複しない番号を入力する。
- 提出日、有効期限、金額、粗利率、ステータスを入力する。
- 商品と在庫をできるだけ紐づける。
- 支払条件、納品条件、備考は見積PDFに出しても問題ない表現にする。
- 失注時は失注理由を残す。

### 推奨例

| 項目 | 推奨 |
| --- | --- |
| 見積番号 | Q-20260705-001 |
| 通貨 | JPY |
| ステータス | 提出済 |
| 有効期限 | 2026-07-31 |

### 非推奨例

| 非推奨 | 理由 |
| --- | --- |
| 見積A | 重複しやすい |
| 未定 | 金額集計できない |
| たぶん採用 | ステータスとして曖昧 |

## ステータス

### 顧客・案件ステータス

以下の候補を優先する。

- 未接触
- 送信済
- 返信あり
- 商談中
- 見積提出
- 成約
- 失注

### 入力ルール

- 状況が変わったらすぐ更新する。
- 成約、失注は最終状態として扱う。
- 詳細な状況はステータスではなくメモや商談履歴に書く。

### 非推奨ステータス

| 非推奨 | 理由 |
| --- | --- |
| いい感じ | 集計できない |
| 保留っぽい | 判断基準が曖昧 |
| 山田さん確認中 | 担当者名と状態が混ざる |

## タグ

### 入力ルール

- タグは短い名詞にする。
- 1タグ1意味にする。
- 同じ意味のタグを増やさない。
- タグに長文を書かない。
- 複数タグは用途ごとに分ける。

### 推奨タグ例

| 分類 | 推奨タグ |
| --- | --- |
| 商品 | 牛肉, チーズ, ベーコン, 水産 |
| 温度帯 | 冷凍, 冷蔵, 常温 |
| 顧客特徴 | 高級, 輸入, 差別化, 量販 |
| 営業状況 | サンプル中, 見積中, 要フォロー |
| 注意 | NG, 支払注意, クレーム注意 |

### 非推奨タグ例

| 非推奨 | 理由 |
| --- | --- |
| 高級商品に興味がありそう | 長すぎる。メモ向き |
| ﾁｰｽﾞ | 表記ゆれ |
| Cheese | 日本語タグと混在 |
| 山田さん | 担当者名はタグにしない |

## 表記ゆれ防止ルール

### 法人格

| 推奨 | 非推奨 |
| --- | --- |
| 株式会社 | （株）, ㈱ |
| 有限会社 | （有）, ㈲ |
| 合同会社 | LLCだけ |

### 数値

| 推奨 | 非推奨 |
| --- | --- |
| 100 | 約100 |
| 1200 | 1,200円/kgを数値欄へそのまま |
| 20% | 20パー |

### 温度帯

| 推奨 | 非推奨 |
| --- | --- |
| 冷凍 | Frozen, 凍結 |
| 冷蔵 | Chill, チルド |
| 常温 | ドライ, ambient |

## 運用ルール

- 新しい表記を作る前に、既存の顧客・商品・タグを検索する。
- 判断に迷った場合は、正式情報を優先し、補足はメモに残す。
- 商談履歴やクレーム履歴は削除せず、訂正・補足で残す。
- AIメールやAI分析に使う情報は、誤字や略称を減らして入力する。
- 定期的にタグとステータスの棚卸しを行う。

## 最後

このルールは検索、集計、ダッシュボード、AI生成、引き継ぎ品質を守るための基準です。

今後、新しい入力項目を追加した場合は、この `DATA_RULE.md` も更新してください。
